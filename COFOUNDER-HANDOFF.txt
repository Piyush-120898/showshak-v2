───────────────────────────────────────────────────────────────
  1. WHAT SHOWSHAK IS (one breath)
───────────────────────────────────────────────────────────────
  A cross-platform STREAMING DISCOVERY layer — NOT a streaming
  service, NOT social media. Curators post short vertical CLIPS
  recommending a show/movie; users feel something, trust the
  curator, tap "WATCH IT" → go to the streaming platform.

  Category we own: "Trust-based, creator-curated, action-oriented
  discovery." = Letterboxd's soul + TikTok's format + an action layer.

  The core loop:
    Curator posts a clip → user watches & trusts → taps Watch It
    → lands on Netflix/Prime/etc.

  We are building the WEBSITE first as the reference for the future
  NATIVE APP (founder knows web, not mobile yet).


───────────────────────────────────────────────────────────────
  2. SACRED PRODUCT PHILOSOPHY (never violate)
───────────────────────────────────────────────────────────────
  - CREATOR FIRST, SHOW SECOND. The clip never shows the title
    upfront — the title is revealed ONLY in the Watch It sheet.
    (If they've seen it, they skip; trust the curator first.)
  - FIRE (🔥) = the like. Not a heart. "This is lit."
  - WATCH IT = the climax. Never "Watch Now". Must feel
    frictionless and PRIVATE — never make the user feel their
    Watch It taps are tracked publicly.
  - HIDE THE SCOREBOARD: public surfaces show ONLY content signals
    (per-clip fires, now per-clip VIEWS) + creator + followers/clips.
    NEVER show fires-received total or Watch-It taps publicly —
    those are the curator's PRIVATE cockpit only, enforced in the DB
    by Row-Level Security (not just UI).
  - CURATORS, not influencers. Taste authorities.
  - NO open hashtags. Structured signals only (show pick + fixed
    vibe set + caption mining on our side).
  - Familiar interactions, ShowShak soul — never "just another
    TikTok/Netflix".

───────────────────────────────────────────────────────────────

  BACKEND: Supabase (Postgres + Auth + Storage + Edge Functions,
    Mumbai region). Migrations live in supabase/migrations/ and are
    APPLIED MANUALLY by the founder in the Supabase SQL editor.
    - Mux for video (HLS via <mux-player>, public playback IDs).
      NOW ON PAY-AS-YOU-GO (upgraded this session) — unlimited
      storage + $20/mo usage credit.
    - TMDB for titles + watch-providers. Now fetched SERVER-SIDE by the
      tmdb-providers Edge Function (search/link/manual/batch) — TMDB is
      DNS-blocked on Indian ISPs but reachable from Supabase's servers, so the
      old local data/ingest-tmdb.js VPN run is no longer required. Browser
      NEVER touches TMDB (it reads cached titles.providers, or proxies search
      through the function). The TMDB key is a function secret.

  GOLDEN BACKEND RULES:
    - Counts are DERIVED from event rows; *_count columns are caches.
    - Media → Mux/Storage; DB stores only IDs/URLs, never bytes.
    - Privacy enforced by RLS, not UI.
    - Soft deletes (deleted_at) everywhere; created_at everywhere.

  HOSTING: GitHub Pages (repo: github.com/Piyush-120898/showshak-v2,
    branch main). Pushing to main auto-rebuilds the live page.
